# Faraway 接口文档

本文档从 `开发文档.md` 中提取并整理，面向后端联调使用。

## 1. 通用约定

### 1.1 接口风格

- 前端统一通过 HTTP API 调用后端
- 前端请求层会统一注入 token、处理超时、错误提示、响应解包
- 接口字段统一使用后端返回字段，不在页面层临时重命名
- 前端当前采用分页参数：`page`、`pageSize`

### 1.2 鉴权

- 登录态通过 `token` 维护
- 本地会缓存：
  - `token`
  - `refresh_token`
  - `user_info`
- 除登录、验证码、公开内容接口外，其余接口默认按需鉴权

### 1.3 上传约定

- 当前项目统一走“后端直传接口”
- 前端直接把文件提交给 Faraway 自建后端
- 后端负责文件校验、存储、返回最终 URL
- 本期不采用“后端签发上传凭证”方案
- 图片上传、视频上传、封面图处理统一走后端上传接口

### 1.4 响应格式

原开发文档没有完全写死统一外层结构，但前端 `api/request.js` 需要统一解包。建议后端统一使用：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

说明：

- `data` 内放具体业务数据
- 列表接口建议统一返回 `page`、`pageSize`、`total`、`list`
- 若后端已有既定包装格式，也请全接口保持一致

- 方法：`GET`
- 路径：`/api/search`

请求参数：

- `keyword`：搜索关键词
- `type`：`all | strategy | vlog`

- `pageSize`

{

  "type": "all",
  "page": 1,
  "pageSize": 10,
  "total": 23,
  "strategyList": [],
  "vlogList": []
}

#### `getStrategyList(params)`

- 建议方法/路径：`GET /api/strategies`
- 请求参数建议：
  - `keyword`

  - `page`

#### `getStrategyDetail(id)`

- 建议方法/路径：`GET /api/strategies/{id}`
- 返回建议补充前端状态字段：
  - `isLiked`
  - `isFavorited`
  - `commentCount`
  - `shareCount`

#### `createStrategy(payload)`

- 建议方法/路径：`POST /api/strategies`
- 语义说明：
  - 该接口用于“正式发布攻略”
  - AI 做攻略页点击“正式保存为攻略”时调用这个接口
  - 调用成功后，该攻略应作为正式内容进入“我的发布”
- 请求体建议：

```json
{
  "title": "string",
  "summary": "string",
  "content": "string",
  "destination": "string",
  "days": 5,
  "coverUrl": "https://...",
  "tags": ["string"],
  "status": "published"
}
```

#### `saveStrategyDraft(payload)`

- 建议方法/路径：`POST /api/strategies/drafts`
- 语义说明：
  - 该接口用于“保存攻略草稿”
  - AI 做攻略页点击“保存草稿”时调用这个接口
  - 草稿只进入草稿箱，不进入“我的发布”
- 请求体与创建攻略基本一致，`status` 为 `draft`

## 5. 前端重点依赖
